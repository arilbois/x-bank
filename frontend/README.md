# ContentBank v2 — Frontend

React + TanStack Router/Query + Tailwind UI for the ContentBank v2 Go backend.

## Stack
- Vite 5
- React 18 + TypeScript
- TanStack Router (file-based)
- TanStack Query
- Tailwind CSS 3
- shadcn/ui-style primitives

## Local dev

```bash
npm install
npm run dev
```

The dev server runs on http://localhost:5173 and proxies `/api/*` to the Go backend at http://localhost:8080.

Make sure the Go server is running first (see `../cmd/server` or just `go run ./cmd/server` from the project root).

## Build

```bash
npm run build
```

Output goes to `dist/`.

## Routes

| Path | Description |
| --- | --- |
| `/login` | Username + password sign-in |
| `/` | Dashboard (totals per category + recent articles) |
| `/articles` | Paginated list with category / source / search filters |
| `/articles/:id` | Article detail + AI analysis (when present) |
| `/trending` | Top-scored articles, filter by category |
| `/scrape` | Admin only — trigger manual scraper runs |

All routes except `/login` require an auth token (stored in localStorage as `contentbank_token`).

## API client

`src/lib/api.ts` exposes a pre-configured axios instance. Every request automatically attaches the `Authorization: Bearer *** from `localStorage`. On 401, the token is cleared and the user is redirected to `/login`.

## Configuration

| Env | Default | Purpose |
| --- | --- | --- |
| `VITE_API_BASE_URL` | `/api` | Path prefix for API calls. Vite proxies this to the Go server. |
